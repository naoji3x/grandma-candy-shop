# 080-業務イベント一覧

業務上の重要なイベント（販売、仕入など）を表形式で整理します。

## ドキュメントの目的

- 業務システムで発生する主要イベント（販売、仕入、在庫調整など）を体系的に整理し、設計・開発・テスト・運用の品質を高める。
- 業務フローやシステム機能設計の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 業務イベントの一覧表（イベント名、概要、発生タイミング、関連オブジェクト等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 業務イベントを明確にすることで、設計・開発・テスト・運用の品質と効率を高める。
- 業務フローやシステム機能設計の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- 業務イベントが曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- 業務フローやシステム機能設計の認識ズレによる品質低下・トラブルにつながる。

## サンプル

### 業務イベント一覧（小売業務システム例）

| イベントID             | イベント名 | 何が起きたか                 | いつ         | 誰が／何が      | 発生条件               | 事実データ                 |
| ---------------------- | ---------- | ---------------------------- | ------------ | --------------- | ---------------------- | -------------------------- |
| bev-sale-checkout      | 販売       | 販売の会計が確定された       | レジ会計時   | 店員            | 取引が確定可能な状態   | 取引ID、明細、支払情報     |
| bev-procurement-recv   | 仕入       | 仕入の納品が確定された       | 納品確定時   | 仕入担当/仕入先 | 納品情報が揃っている   | 納品ID、納品明細、仕入先   |
| bev-stock-adjust       | 在庫調整   | 在庫調整（差異）が確定された | 棚卸・破損時 | 担当者          | 差異理由が入力済み     | 商品、現在庫数、差異理由   |
| bev-sale-return-accept | 返品       | 返品が受付された             | 返品受付時   | 店員/顧客       | 対象取引が特定できる   | 取引ID、返品明細、理由     |
| bev-order-placed       | 発注       | 発注が確定された             | 発注確定時   | 店員/システム   | 発注対象が確定している | 発注ID、商品、数量、仕入先 |

---

注：上記は例です。実プロジェクトの業務イベントを具体的に記載してください。
