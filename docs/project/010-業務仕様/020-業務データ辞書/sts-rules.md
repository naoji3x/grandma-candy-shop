# ステータス （Status）作成ルール

業務上の主要なエンティティの状態（ステータス）を一覧で整理します。各ステータスの呼称・場所・管理帳簿・説明などを明記し、業務プロセスにおける状態遷移や管理単位を明確にします。

## 1. メタデータ

| 項目       | 説明                          | 必須 |
| ---------- | ----------------------------- | ---- |
| id         | ステータスID (sts-xxx-xxxx)   | ○    |
| type       | `domain` 固定                 | ○    |
| title      | ステータス名                  | ○    |
| status     | `draft`/`ready`/`deprecated`  | ○    |
| version    | バージョン                    | 任意 |
| owners     | 担当者                        | 任意 |
| tags       | タグ・分類                    | 任意 |
| supersedes | 置き換え関係（古仕様→新仕様） | 任意 |

## 2. 記述ルール

- エンティティのステータスを表形式で整理する。
- ステータス、呼称、説明などの項目を含めるとともに、必要に応じて補足項目を追加する。
- 意味が曖昧な説明や略語のみの記載は避け、簡潔かつ明確に記載する。

## 3. サンプル

| ステータス   | 呼称         | 場所             | 主な業務       |
| ------------ | ------------ | ---------------- | -------------- |
| 仕入前       | 納入商品     | 仕入前           | 受入           |
| 仕入直後     | 在庫商品     | バックヤード     | 受入、在庫保管 |
| 検品不合格   | 返品商品     | バックヤード     | 検品           |
| 販売可能状態 | 販売商品     | 売り場棚         | 品出し、陳列   |
| 購買受付中   | 会計対象商品 | レジまわり       | 会計           |
| 購買済引渡前 | 引渡前商品   | レジまわり       | 会計           |
| 販売完了     | 販売済商品   | お客様へ提供済み | 商品引渡し     |
| 後払い扱い   | 掛売商品     | つけ帳           | つけ回収       |
